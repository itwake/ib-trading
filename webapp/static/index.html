这个系统是什么？

一套全自动执行的美股短线策略：收盘前筛选当日大跌的中大盘股 → 收盘竞价（MOC）买入 → 当晚起挂 +1.5% 限价卖出 → 次日按配置时点改挂追踪卖出，当日必清仓（2026-07-19 起默认 10:00 挂 0.5%——依据实盘出场阶段分析）。 开不开仓由「环境闸门」决定（VIX 或大盘跌幅达标才做，依据是全年真实成交审计：恐慌日的反弹显著可靠，平静日的个股暴跌容易继续跌）。 所有动作按交易所日历自动触发，每一步都推送 Discord。

一个交易日的完整流程（美东时间）

15:33 闸门判定 → 15:38 选股与预算分配 → 15:45 提交 MOC 买单（收盘价成交）→ 16:10 成交入台账 → 挂隔夜限价卖单（IBKR 隔夜时段 20:00~次日3:50，默认 20:05）→ 挂盘前限价（默认 04:05）→ 撤限价改挂追踪卖出（时点/宽度可配，当前 10:00 挂 0.5%）→ 午间对账（默认 11:55，固化上午成交）→ 16:20 对账+日报。卖出链时点可在「配置 → 执行时点」按相对锚点的分钟偏移调整。
没到 +1.5% 也会被追踪单在当天卖掉——系统永远不留隔夜库存超过一晚（除非停牌等异常，会告警）。

快速上手（当前是 dry 演练模式）

1️⃣ 看「总览」：确认网关正常（绿色）、有净值快照、事件表时间合理；
2️⃣ 交易日盯 Discord：每个自动动作都会播报，对照「事件表」检查是否按点执行；
3️⃣ 看「每晚决策」：确认闸门判定和你的预期一致（VIX/SPY 数值都有记录）；
4️⃣ dry 跑顺 2~3 个交易日后 →「配置」页把 mode 切成 live（需输入 LIVE）→ 建议先把「每晚预算」调小（如 5000）跑一周 → 再逐步加回；
5️⃣ 任何时候想停：右上角「⏸ 暂停开仓」（持仓照常卖出）；想彻底跑路：「手动操作」页一键清仓。

lot 状态含义（交易台账/持仓表里的"状态"列）

FILLED 已买入，还没挂卖单 → OVERNIGHT 隔夜限价卖单已挂 → PREMARKET 盘前限价已挂 → TRAILING 开盘后追踪卖单已挂 → CLOSED 已平仓（出场价和盈亏来自 IB 真实成交回填）。

常见问题

页面某处显示"缓存 @ 时间"？ 该数据源（如 Yahoo 行情）暂时取不到，系统自动显示最后一次成功的值并标注时间，不影响交易（交易走 IB 通道）。
网关显示无响应？ 检查家中 Windows：① 隧道任务在跑吗；② Gateway 登录着吗。守护进程会持续告警直到恢复；期间自动买入会失败并播报，已挂在交易所的卖单不受影响。
今晚为什么没买？ 看「每晚决策」：要么闸门拦截（VIX/SPY 未达标），要么可用资金低于下限，要么你按了暂停。
闸门太严/太松？「配置」页调 VIX 阈值：17 折中（历史约 40~75% 的晚上放行）、19 保守（约 25~50%）。
候选追踪是什么？ 每晚把跌幅榜前 N 名（默认 20，含没买的）全部登记，次日自动回填「如果买了会怎样」：命中率、收益按名次段和板块聚合（每晚决策页），用真实数据回答"该买第几名到第几名""哪些板块的大跌不该接"。
出场价带 ⚠️估价 是什么意思？ 该 lot 的持仓已消失但查不到卖出成交明细（如网关重启丢记录），系统按当时行情估价入账并告警，请用 IBKR 成交记录或 Flex 报告核对修正（deploy/repair_lots.py）。
怎么彻底停止系统？ 服务器上 systemctl stop autotrader；或者面板「暂停开仓」+ 手动清仓，守护进程留着继续做对账和告警。
我自己在 TWS 手动挂了卖单，系统会冲突吗？ 不会。系统挂单前会统计该票全部在途卖单（包括你手动挂的），可卖数量=持仓−在途，不足就缩量或跳过（防超卖）；系统撤单时只撤自己打了 autotrader 标记的单，你的手动单永远不动。
股票已经涨超 +1.5% 了还按目标价挂？ 不会。挂单前会查实时行情，市价高于目标时按市价×(1+0.1%) 跟随抬价（不超过卖一价），见配置页「跟随市场抬价」。

术语表

MOC＝Market-On-Close，收盘竞价市价单，按官方收盘价成交；OVERNIGHT＝IBKR 隔夜交易时段（美东 20:00~3:50，仅限价单，单场次有效）； TRAIL＝追踪卖出：价格从最高点回落配置比例即触发；VIX＝恐慌指数，越高市场越动荡； NetLiq＝净清算价值（现金+持仓）；可用资金＝还能动用的保证金余量，太低有强平风险； lot＝一笔买入形成的一个仓位单元，各自独立管理目标价。
