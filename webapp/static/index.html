这个系统是什么？

一套全自动执行的美股短线策略：收盘前筛选当日大跌的中大盘股 → 收盘竞价（MOC）买入 → 当晚起挂 +1.5% 限价卖出 → 次日开盘后改 0.3% 追踪卖出，当日必清仓。 开不开仓由「环境闸门」决定（VIX 或大盘跌幅达标才做，依据是全年真实成交审计：恐慌日的反弹显著可靠，平静日的个股暴跌容易继续跌）。 所有动作按交易所日历自动触发，每一步都推送 Discord。

一个交易日的完整流程（美东时间）

15:33 闸门判定 → 15:38 选股与预算分配 → 15:45 提交 MOC 买单（收盘价成交）→ 16:10 成交入台账 → 20:05 挂隔夜限价卖单（+1.5%，IBKR 隔夜时段 20:00~次日3:50）→ 次日 04:05 挂盘前限价 → 09:31 撤限价改 0.3% 追踪卖出 → 16:20 对账+日报。
没到 +1.5% 也会被追踪单在当天卖掉——系统永远不留隔夜库存超过一晚（除非停牌等异常，会告警）。

快速上手（当前是 dry 演练模式）

1️⃣ 看「总览」：确认网关正常（绿色）、有净值快照、事件表时间合理；
2️⃣ 交易日盯 Discord：每个自动动作都会播报，对照「事件表」检查是否按点执行；
3️⃣ 看「每晚决策」：确认闸门判定和你的预期一致（VIX/SPY 数值都有记录）；
4️⃣ dry 跑顺 2~3 个交易日后 →「配置」页把 mode 切成 live（需输入 LIVE）→ 建议先把「每晚预算」调小（如 5000）跑一周 → 再逐步加回；
5️⃣ 任何时候想停：右上角「⏸ 暂停开仓」（持仓照常卖出）；想彻底跑路：「手动操作」页一键清仓。

lot 状态含义（交易台账/持仓表里的"状态"列）

FILLED 已买入，还没挂卖单 → OVERNIGHT 隔夜限价卖单已挂 → PREMARKET 盘前限价已挂 → TRAILING 开盘后追踪卖单已挂 → CLOSED 已平仓（出场价和盈亏来自 IB 真实成交回填）。

常见问题

页面某处显示"缓存 @ 时间"？ 该数据源（如 Yahoo 行情）暂时取不到，系统自动显示最后一次成功的值并标注时间，不影响交易（交易走 IB 通道）。
网关显示无响应？ 检查家中 Windows：① 隧道任务在跑吗；② Gateway 登录着吗。守护进程会持续告警直到恢复；期间自动买入会失败并播报，已挂在交易所的卖单不受影响。
今晚为什么没买？ 看「每晚决策」：要么闸门拦截（VIX/SPY 未达标），要么可用资金低于下限，要么你按了暂停。
闸门太严/太松？「配置」页调 VIX 阈值：17 折中（历史约 40~75% 的晚上放行）、19 保守（约 25~50%）。
影子 T+1/T+2 是什么？ 每笔真实平仓后，系统自动计算"如果按 T+1/T+2 收盘强制离场会赚多少"，攒一个月数据就能用实盘对比三种出场规则（交易台账页顶部）。
怎么彻底停止系统？ 服务器上 systemctl stop autotrader；或者面板「暂停开仓」+ 手动清仓，守护进程留着继续做对账和告警。

术语表

MOC＝Market-On-Close，收盘竞价市价单，按官方收盘价成交；OVERNIGHT＝IBKR 隔夜交易时段（美东 20:00~3:50，仅限价单，单场次有效）； TRAIL 0.3%＝追踪卖出：价格从最高点回落 0.3% 即触发；VIX＝恐慌指数，越高市场越动荡； NetLiq＝净清算价值（现金+持仓）；可用资金＝还能动用的保证金余量，太低有强平风险； lot＝一笔买入形成的一个仓位单元，各自独立管理目标价。
