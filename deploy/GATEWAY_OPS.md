# IB Gateway 运维手册（192.168.8.221）

背景：两周内两台 Gateway 各出过一次事故——.223 于 2026-07-07 JVM 内存溢出崩溃
（当晚整晚停摆，hs_err_pid7712.log 显示运行仅 1h46m 后 OOM）；.221 于 07-10、07-15
两次心跳无响应告警。Gateway 是系统目前唯一无法自愈的单点。以下操作按收益排序，
RDP 到 192.168.8.221 执行（此机无 SSH，无法远程自动化）。

## 1. 调大 JVM 内存（防 OOM 崩溃，5 分钟）

编辑 `C:\Jts\ibgateway\<版本号>\ibgateway.vmoptions`（或桌面快捷方式同名文件）：

```
-Xmx1536m
```

默认多为 768m；.223 的崩溃正是堆耗尽。改完重启 Gateway 生效。

## 2. 登录时区改美东（消除成交查询窗口错位，5 分钟）

Gateway 登录界面 → More Options → Time Zone → **America/New_York**。

原因：reqExecutions 只返回"登录时区当日午夜以来"的成交，上海时区的午夜=12:00 ET，
曾导致上午成交在日报对账时不可见（15 手被 0 价强平的事故根因）。代码层已用
11:55 ET 午间对账兜底，此改动是双保险。注意：改时区可能改变 Gateway 每日
自动重启的触发时刻，改完看一眼第 3 条。

## 3. 每日自动重启挪到安全时段（5 分钟）

Configure → Settings → Lock and Exit → Auto restart，时间设为 **03:52 America/New_York**。

依据：系统时刻表里 03:50 ET 隔夜时段结束、04:00 ET 挂盘前单——03:52~03:58 是全天
唯一没有任何在途动作的窗口。切勿设在 09:30~16:20 ET（盘中重启会丢当日成交记录窗口）。

## 4. Trusted IPs 确认

API → Settings → Trusted IPs 须含 **192.168.8.237**（服务器直连）。

## 4b. Master API Client ID 设为 31（跨客户端撤单授权，5 分钟）

API → Settings → **Master API client ID = 31**。

原因：IB 规定普通 API 客户端只能撤自己下的单；面板手动操作用 clientId 33 下的单，
守护进程（31）撤不掉（2026-07-20 实例：10:00 换追踪时 4 只被跳过，需手动重跑）。
把 31 设为 Master 后，守护进程可以看到并撤销所有客户端的单，手动兜底和自动链
从此无缝衔接。代码侧的撤单+轮询验证（740072d）已就位，此设置是最后一块拼图。

## 5. 进阶（可选）：IBC 自动拉起

IBC (https://github.com/IbcAlpha/IBC) 可在 Gateway 进程死掉时自动重启并自动登录
（2FA 仍需每周手机确认一次）。配置文件样例见 IBC 文档 + Windows 任务计划程序
"登录时启动"。等 1~3 条跑稳后再考虑；届时服务器的心跳告警（已有）负责发现，
IBC 负责拉起，形成闭环。

## 服务器侧已有的监控（无需操作）

- 每 10 分钟握手探测，状态变化即 Discord 告警（daemon heartbeat）
- 买入链步骤失败单独 critical 告警
- 面板顶栏实时网关状态灯
